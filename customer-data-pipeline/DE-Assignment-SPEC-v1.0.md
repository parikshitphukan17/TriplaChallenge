### **1\. CustomerProfile Entity**

This entity is a master record representing a unique individual customer, consolidated through identity resolution. It is stateful and represents the "current" state of the customer profile.

source\_type and source\_id as external keys in our master. The schema for this entity integrates requirements for customer profiles with information commonly available from source systems. Notably, fields like

emails, phone\_numbers, and addresses are defined as arrays of objects, natively supporting one-to-many relationships.

### **2\. Membership Entity**

This entity is a record that represents how a customer belongs to a specific loyalty program. A single customer can have multiple membership records.

Membership Field Specification

| Field Name | Data Type | Req. | Description / Constraints | Source Mapping Guide | Example |
| :---- | :---- | :---- | :---- | :---- | :---- |
| source\_type | string | Y | Source system name associated with the individual or corporate profile. opera, nehops, etc. | System name like opera, nehops, table\_check, bv\_manager | “opera” |
| source\_id | string | Y | Source identifier associated with the individual.  Required for individual members. | Profile ID, customer number, corporate ID, etc. | “12345” |
| program\_id | integer | Y | Unique identifier of the membership program. | Program ID from the source system. | 145 |
| program\_name | string | Y | Name of the membership program | Membership program name | "Best loyalty" |
| membership\_id | string | Y | The member ID within that program. | Program-specific member ID. | "1234567890" |
| rank\_name | string | Y | Member rank (e.g., "Gold", "Silver"). | Member rank. | "Gold" |
| created\_at | string (date) | Y | Date of enrollment. YYYY-MM-DD format. | Enrollment date. | "2022-10-01" |
| status | string (enum) | Y | Status of member. Follow the enum values: active, inactive. Only records with `status = 'active'` are eligible for downstream sync. Records with `status = 'inactive'` must be excluded. | Member status. | "active" |
| etl\_metadata | object | Y | Metadata about record generation. Structure: {"source\_system": string, "source\_file": string, "processed\_at": datetime} | Generated on the ETL side. | {"source\_system": "opera",...} |
